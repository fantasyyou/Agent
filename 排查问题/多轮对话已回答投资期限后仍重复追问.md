# 多轮对话已回答投资期限后仍重复追问

对应截图：[多轮对话已回答投资期限后仍重复追问.png](./多轮对话已回答投资期限后仍重复追问.png)

## 问题现象

用户按照客服提示依次提供了投资期限和投资金额：

```text
客服：这笔资金预计多长时间内不会使用？
用户：从现在算，未来3个月基本不会动用。

客服：您计划投入的大致金额是多少？
用户：20万。

客服：这笔资金预计多长时间内不会使用？
```

系统已经询问过投资期限，用户也作出了回答，但在用户继续提供金额后，系统又重复询问投资期限。

## 排查证据

查询 Redis 中的短期对话状态：

```bash
docker exec \
  -e REDISCLI_AUTH=redis \
  agent-redis \
  redis-cli GET \
  'agent:dialogue:usr_7301e1ba1ffd441862f9f4b953cf7858:web-1784298352995-0.5370394593826645'
```

实际结果：

```json
{
  "intent": "product_recommendation",
  "slots": {
    "liquidity_requirement": "low",
    "risk_tolerance": "conservative"
  },
  "status": "need_clarification",
  "last_question": "这笔资金预计多长时间内不会使用？"
}
```

状态中缺少：

```json
{
  "investment_period_months": 3,
  "investment_amount": 200000
}
```

这说明 Redis 没有主动丢失数据，而是 Python 在将状态写入 Redis 之前，没有成功保留期限和金额。

## 根本原因

问题由两个因素共同造成。

### 1. 数值类型校验过于严格

用户表达的金额和期限可能被模型或 protobuf 转换为不同形式：

```text
3个月、3、3.0
20万、20万元、200000、200000.0
```

原代码要求投资期限必须是 Python 的 `int`。protobuf `Struct` 中的数字经过跨服务传输后可能变成 `float`，因此数值为整数的 `3.0` 也会被判定为无效。

模型如果返回字符串 `"3个月"` 或 `"20万"`，同样会被原有类型检查过滤。

### 2. 避免重复提问的策略掩盖了提取失败

原策略会大幅降低上一问题的分数：

```text
期限没有提取成功
→ 期限仍然缺失
→ 为避免重复期限问题，系统改问金额
→ 金额也没有提取成功
→ 系统又改回询问期限
```

因此界面表现成问题在期限和金额之间来回跳转。

## 修复前的关键代码

### 数值校验

文件：`agent-python/service/requirement_analysis_service.py`

```python
def _valid_slot_value(definition: SlotDefinition, value) -> bool:
    """按工作流定义校验模型提取值，避免仅凭字段存在就进入执行流程。"""

    if value in (None, "", []):
        return False
    if definition.value_type == SLOT_VALUE_TYPE_STRING:
        valid_type = isinstance(value, str)
    elif definition.value_type == SLOT_VALUE_TYPE_NUMBER:
        valid_type = (
            isinstance(value, (int, float))
            and not isinstance(value, bool)
            and value > 0
        )
    elif definition.value_type == SLOT_VALUE_TYPE_INTEGER:
        valid_type = (
            isinstance(value, int)
            and not isinstance(value, bool)
            and value > 0
        )
    else:
        return False
    return valid_type and (
        not definition.allowed_values or value in definition.allowed_values
    )
```

主要问题：

- `3.0` 不是 `int`，投资期限校验失败；
- `"3个月"` 无法转换为整数；
- `"20万"` 无法转换为人民币元数值；
- 代码只做校验，没有先进行标准化。

## 解决思路

修复采用三层控制：

1. 提示模型按照标准类型返回金额和期限；
2. Python 对跨服务数字和常见中文金额、期限表达进行确定性标准化；
3. 上一问题仍未得到有效值时，继续澄清同一个字段，不再静默跳题。

标准化示例：

```text
3个月     → 3
3.0       → 3
半年      → 6
1年       → 12
20万      → 200000
20万元    → 200000
200000元  → 200000
```

```python
def _normalize_slot_value(definition: SlotDefinition, value) -> tuple[bool, object]:
    """校验并规范化模型或跨服务状态中的参数值。"""

    if value in (None, "", []):
        return False, value
    if definition.value_type == SLOT_VALUE_TYPE_STRING:
        normalized = value.strip() if isinstance(value, str) else value
        valid_type = isinstance(normalized, str) and bool(normalized)
    elif definition.value_type == SLOT_VALUE_TYPE_NUMBER:
        normalized = _parse_positive_number(value)
        valid_type = normalized is not None
    elif definition.value_type == SLOT_VALUE_TYPE_INTEGER:
        normalized = _parse_positive_integer(value)
        valid_type = normalized is not None
    else:
        return False, value
    if not valid_type or (
        definition.allowed_values and normalized not in definition.allowed_values
    ):
        return False, value
    return True, normalized
```

### 兼容整数浮点数和常见期限表达

```python
def _parse_positive_integer(value) -> int | None:
    """接受protobuf产生的3.0及“3个月”“半年”“1年”等期限表达。"""

    if isinstance(value, bool):
        return None
    if isinstance(value, (int, float)):
        number = float(value)
    elif isinstance(value, str):
        text = value.strip().replace(" ", "")
        if text == "半年":
            return 6
        year_match = re.fullmatch(r"(\d+(?:\.\d+)?)年", text)
        if year_match:
            number = float(year_match.group(1)) * 12
        else:
            month_match = re.fullmatch(r"(\d+(?:\.\d+)?)(?:个?月)?", text)
            if not month_match:
                return None
            number = float(month_match.group(1))
    else:
        return None
    if not math.isfinite(number) or number <= 0 or not number.is_integer():
        return None
    return int(number)
```

### 兼容中文金额单位

```python
def _parse_positive_number(value) -> int | float | None:
    """接受普通数字和“20万”“200000元”等常见人民币金额表达。"""

    if isinstance(value, bool):
        return None
    if isinstance(value, (int, float)):
        number = float(value)
    elif isinstance(value, str):
        text = value.strip().replace(",", "").replace("，", "").replace(" ", "")
        match = re.fullmatch(
            r"(?:人民币)?([0-9]+(?:\.[0-9]+)?)([千万元亿]?)(?:人民币)?(?:元)?(?:左右)?",
            text,
        )
        if not match:
            return None
        multiplier = {
            "": 1,
            "千": 1_000,
            "万": 10_000,
            "万元": 10_000,
            "亿": 100_000_000,
        }[match.group(2)]
        number = float(match.group(1)) * multiplier
    else:
        return None
    if not math.isfinite(number) or number <= 0:
        return None
    return int(number) if number.is_integer() else number
```

### 上一字段仍缺失时继续澄清

文件：`agent-python/service/question_policy.py`

```python
for candidate in candidates:
    if last_question and last_question == candidate.question:
        return candidate
```

含义是：

- 上一问题对应的字段仍然缺失：继续询问或澄清该字段；
- 上一字段取得有效值：它会从缺失列表移除，再选择下一个字段。
